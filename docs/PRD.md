# 소개팅모아 - Product Requirements Document (PRD)

## 1. 서비스 개요

### 서비스명
**소개팅모아** (가칭 / 추후 변경 가능)

### 한 줄 정의
> 전국 로테이션 소개팅 업체의 일정, 가격, 잔여석을 한곳에서 모아보고 바로 신청할 수 있는 앱

### 배경 및 문제 정의
- 2030 직장인 사이에서 로테이션 소개팅이 빠르게 확산 중 (한 회 3~5만원, 최대 20명과 대화)
- 관련 업체가 우후죽순 생겨나고 있으나 정보가 각 업체 사이트/인스타그램에 분산되어 있음
- 사용자는 러브매칭, 연인어때, 에모셔널오렌지, 프립, 문토 등을 개별적으로 방문해야 함
- **→ 통합 탐색 플랫폼이 없다 = 해결할 문제**

### 레퍼런스 서비스
- **모아뷰 (moaview.co.kr)**: 다수의 체험단 사이트 정보를 크롤링하여 한곳에서 필터/검색/모아보기 제공
  - 참고 포인트: 마감순 정렬, 경쟁률 필터, 마감 임박 뱃지, 한방에 로딩 구조
  - 차이점: 우리는 모바일 네이티브 앱 + 20~30대 타겟 최신 UX

## 2. 타겟 사용자

### Primary Target
- **연령**: 25~35세
- **성별**: 남녀 모두
- **특징**:
  - 바쁜 직장인으로 이성 만날 기회가 없음
  - 앱/디지털 환경에 익숙함
  - 가성비 중시 (같은 돈에 여러 명 만나고 싶음)
  - 결혼정보회사는 부담스럽고 데이팅앱은 신뢰도 낮다고 느낌
  - 정보 탐색보다 빠른 결정을 선호

### Secondary Target
- 로테이션 소개팅 주최 업체 (나중에 직접 등록 기능 추가 시)

### 사용자 Pain Point
1. 원하는 날짜/지역에 맞는 소개팅 찾기 위해 여러 사이트를 돌아다녀야 함
2. 인스타그램 DM으로만 신청 가능한 업체가 많아 불편
3. 잔여석/마감 여부를 실시간으로 확인하기 어려움
4. 업체 신뢰도/후기를 객관적으로 비교하기 어려움
5. 마음에 드는 조건의 일정이 생기면 알림을 받고 싶음

## 3. 핵심 기능 정의

### 3.1 MVP 기능 (v1.0)

#### 📋 이벤트 피드 (홈)
- 전체 업체의 로테이션 소개팅 일정 카드 형태로 통합 표시
- 카드당 표시 정보:
  - 썸네일 이미지 1~2장
  - 업체명 + 업체 로고
  - 이벤트 제목
  - 날짜 / 요일
  - 지역 (강남, 홍대, 수원 등)
  - 가격 (남/여 구분)
  - 성비 구성 (예: 6:6, 8:8)
  - 테마 태그 (와인, 커피, 에세이 등)
  - 잔여석 표시 (가능한 경우)
  - 마감 임박 뱃지 (D-3 이하)
  - [신청하기] 버튼 → 업체 페이지 아웃링크
- 정렬: 날짜순 (기본) / 마감임박순 / 최신등록순
- 무한 스크롤

#### 🔍 필터 & 검색
- 지역 필터 (서울 전체 / 강남 / 홍대 / 신촌 / 을지로 / 수원 / 인천 / 대전 등)
- 날짜 필터 (오늘 / 이번주 / 이번달 / 날짜직접선택)
- 가격 필터 (3만원 이하 / 3~5만원 / 5만원 이상)
- 성비 필터 (5:5 / 6:6 / 8:8 이상)
- 테마 필터 (와인 / 커피 / 에세이 / 전시 / 기타)
- 복합 필터 동시 적용 가능
- 최근 검색/필터 5개 저장 (모아뷰 참고)

#### 🔔 알림 설정
- 조건 저장 (지역 + 날짜 + 가격대 + 테마 조합)
- 조건에 맞는 새 일정 등록 시 푸시 알림 발송
- 관심 업체 팔로우 → 해당 업체 새 일정 알림
- 마감 임박 알림 (관심 등록한 이벤트 D-1)

#### 📄 이벤트 상세
- 썸네일 이미지 (최대 5장 슬라이드)
- 업체 정보 (이름, 로고, 운영 지역)
- 이벤트 상세 설명
- 참가 조건 (나이 범위, 직업 조건 등)
- 진행 방식 (1:1 / 2:2 / 로테이션 등)
- [신청하기] CTA → 업체 원본 페이지 아웃링크 (인앱 브라우저)

#### 🏢 업체 페이지
- 업체별 진행 이벤트 목록
- 업체 소개
- 운영 지역/연락처

### 3.2 v2.0 기능 (추후)
- 업체 직접 등록/관리 (업체가 먼저 요청 시)
- 찜 목록
- 참여 후기 (사용자 작성)
- 지도 기반 탐색
- 업체 비교 기능

### 3.3 MVP 제외 기능 (Out of Scope)
- 자체 결제/예약 시스템 (아웃링크로 대체)
- 회원가입/로그인 (푸시 알림 위해서만 익명 토큰 사용)
- 채팅/매칭 기능
- 유료 구독

## 4. UX/UI 가이드라인

### 4.1 디자인 철학
> "앱을 여는 순간, 오늘 밤 설렐 수 있는 곳을 바로 찾는다"

- **즉각성**: 앱 열자마자 오늘/이번주 일정이 바로 보여야 함
- **신뢰감**: 검증된 업체임을 시각적으로 표현
- **설렘**: 차갑지 않은, 따뜻하면서 세련된 무드
- **간결함**: 정보 과부하 없이 필요한 것만

### 4.2 비주얼 방향
- **톤앤무드**: 다크 베이스 or 오프화이트 + 포인트 컬러 (핑크-코랄 or 퍼플 계열)
- **폰트**: 프리텐다드 (한글) / SF Pro (영문)
- **카드 UI**: 둥근 모서리(16px), 그림자 가볍게, 이미지 비율 16:9
- **레퍼런스 앱 무드**: 당근마켓 + 오늘의집 중간 느낌 (정보형이지만 감성적)

### 4.3 핵심 UX 원칙
1. **3초 룰**: 원하는 일정까지 3번 탭 이내에 도달
2. **썸네일 우선**: 텍스트보다 이미지가 먼저 눈에 들어오는 구조
3. **아웃링크 명확화**: "신청은 업체 페이지에서" 라는 것을 명확히 인지시킴
4. **필터 퍼시스턴스**: 앱 종료 후 재진입 시 마지막 필터 유지
5. **로딩 최소화**: 모아뷰처럼 한번에 로딩 후 클라이언트 필터링

### 4.4 화면 목록
```
1. 스플래시 화면
2. 온보딩 (최초 1회 - 관심 지역/테마 설정)
3. 홈 피드 (이벤트 카드 리스트)
4. 이벤트 상세
5. 필터 바텀시트
6. 업체 상세
7. 알림 설정
8. 설정 (앱 버전, 문의, 개인정보처리방침)
```

## 5. 기술 요구사항

### 5.1 앱
- **프레임워크**: Expo (React Native) - iOS/Android 동시 지원
- **상태관리**: Zustand
- **서버 연동**: Supabase JS Client
- **네비게이션**: Expo Router (파일 기반)
- **푸시 알림**: Expo Notifications + Expo Push API
- **아웃링크**: expo-web-browser (인앱 브라우저)

### 5.2 크롤러
- **언어**: Python 3.11+
- **라이브러리**: Playwright (JS 렌더링 대응), BeautifulSoup4, httpx
- **스케줄**: GitHub Actions (매일 오전 6시, 오후 12시, 오후 6시 - 3회)
- **저장**: Supabase REST API

### 5.3 백엔드/DB
- **DB**: Supabase (PostgreSQL)
- **Queue**: pgmq (푸시 알림 비동기 처리)
- **서버리스**: Supabase Edge Functions (Queue 워커)
- **인증**: 없음 (MVP) / Expo 푸시 토큰으로 익명 식별

### 5.4 인프라
- **크롤러 CI/CD**: GitHub Actions
- **앱 빌드/배포**: EAS (Expo Application Services)
- **환경변수**: GitHub Secrets + EAS Secrets

## 8. 제약 사항 및 원칙

1. **자체 결제/예약 없음** - 모든 신청은 업체 페이지 아웃링크로
2. **회원가입 없음 (MVP)** - 마찰 최소화, 푸시 토큰만 사용
3. **업체 직접 등록 없음 (MVP)** - 앱이 성장하면 업체가 먼저 요청할 때 추가
4. **크롤링은 공개 정보만** - 로그인 필요 페이지는 크롤링 안 함
5. **아웃링크 명확화** - 우리 앱에서 결제/개인정보 수집 없음을 명시
